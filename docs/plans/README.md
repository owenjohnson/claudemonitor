# Implementation Plans

This directory contains detailed implementation plans for features and significant changes.

## Purpose

Plans document:
- What will be built
- How it will be implemented
- Key decisions and trade-offs
- Task breakdown and dependencies

## Plan Index

| Name | Status | Description |
|------|--------|-------------|
| - | - | - |

## Creating a Plan

1. Create a new markdown file: `feature-name.md`
2. Include sections for:
   - Overview/Goal
   - Requirements
   - Technical Approach
   - Tasks/Milestones
   - Open Questions
3. Add entry to the index above
4. Review with team before implementation

## Plan Lifecycle

1. **Draft** - Initial proposal
2. **Review** - Under discussion
3. **Approved** - Ready for implementation
4. **In Progress** - Being implemented
5. **Complete** - Implemented and verified
6. **Abandoned** - Not proceeding

## Template

```markdown
# Plan: [Feature Name]

## Status
Draft | Review | Approved | In Progress | Complete | Abandoned

## Overview
Brief description of what this plan accomplishes.

## Requirements
- Requirement 1
- Requirement 2

## Technical Approach
Describe the implementation approach.

## Tasks
- [ ] Task 1
- [ ] Task 2

## Open Questions
- Question 1?
```
